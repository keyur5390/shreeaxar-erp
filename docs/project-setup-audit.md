# Project Setup Audit

This branch is a lightweight, dependency-minimal PHP ERP application. It is not a Laravel install, even though the command entry point is named `artisan` for developer familiarity.

## Working project layout

| Path | Purpose |
| --- | --- |
| `app/` | Application controllers, core routing/database classes, repositories, services, and support helpers. |
| `database/` | SQL migrations, the canonical schema reference, and PHP seeders for default roles, permissions, masters, admin user, quotation statuses, and company settings. |
| `bootstrap/app.php` | Runtime bootstrap, environment loading, and autoload fallback for installs without `vendor/`. |
| `config/` | Application configuration read through helper functions. |
| `resources/views/` | PHP templates for layouts, authentication, dashboard, CRUD pages, quotations, reports, and activity logs. |
| `assets/` | Source CSS and JavaScript. These are intentionally kept at the root as editable source files. |
| `public/` | Web server document root. `public/assets/` is generated by `npm run build` from `assets/`. |
| `storage/` | Runtime SQLite database, uploads, exports, and other generated files. Only `.gitkeep` placeholders are tracked. |
| `docker/` | PHP-FPM, Nginx, and opcode-cache configuration used by Docker Compose. |
| `docs/` | Product blueprint and project setup notes. |
| `tests/run.php` | Smoke tests for migrations, seed data, quotation calculations, and quotation numbering. |

## Stand-alone files reviewed

The root-level files are intentional project entry points rather than random leftovers:

- `artisan` is the CLI runner for migrations, seeding, smoke tests, and setup checks.
- `build-assets.mjs` copies source assets into the public web root.
- `composer.json` defines PHP requirements, autoloading, and script aliases.
- `package.json` defines asset build commands without adding unnecessary frontend dependencies.
- `docker-compose.yml` starts the complete PHP, Nginx, MySQL, Redis, queue, and scheduler stack.
- `.env.example` is the Docker-oriented environment template.

## Validation checklist

Run these commands after cloning or before deployment:

```bash
cp .env.example .env
composer dump-autoload
npm run build
php artisan doctor
php artisan migrate --fresh --seed
php artisan test
```

For Docker, use:

```bash
cp .env.example .env
docker compose build
docker compose up -d
docker compose exec app php artisan doctor
docker compose exec app php artisan migrate --fresh --seed
```
